## 1. 员工创建功能

### 1.1 手动创建员工
| 编号 | 测试场景 | 操作步骤 | 预期结果 | 优先级 |
|------|---------|---------|---------|--------|
| TC-001 | 填写所有必填字段创建员工 | 填写姓名、部门、职位、手机号、邮箱、入职日期、状态，提交 | 创建成功，工号自动生成，跳转详情页 | P0 |
| TC-002 | 必填字段为空提交 | 留空姓名，提交表单 | 表单校验提示"请输入姓名" | P0 |
| TC-003 | 手机号格式校验 | 输入非11位数字手机号 | 提示"请输入正确的手机号" | P0 |
| TC-004 | 邮箱格式校验 | 输入无效邮箱格式 | 提示"请输入正确的邮箱地址" | P0 |
| TC-005 | 填写所有字段（含选填）创建 | 填写全部12个字段，提交 | 创建成功，所有字段正确保存 | P1 |
| TC-006 | 工号自动生成唯一性 | 连续创建两个员工 | 两个工号不同，格式为 EMP-YYYYMMDD-XXX | P0 |

## 验收标准

### 必须通过（P0）
- 所有 P0 测试用例 100% 通过
- 员工 CRUD 完整流程可走通
- 数据持久化正常
- 表单校验完整

### 应该通过（P1）
- P1 测试用例 90% 以上通过
- 搜索和筛选功能正常
- 导入导出基本功能可用

### 可以延后（P2）
- 字段映射、选择导出字段等高级功能
- 响应式布局优化
